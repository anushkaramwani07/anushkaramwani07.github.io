Presenting an image using HTML tag vs presenting an image as a background?

  • While using html tag it will set that image as an element on the page and it has its own height and width automatically
  • It will move with the page as you scroll.
  • If we use it as css background, it doesn't show by itself, its just in the background.
  • The image doesnt have its own size, it fills whatever space the box has.
  • img tag have alt text that is useful for readers.
  • background-img is decorative so it odes not have any alt texts.
  • With css background you can add effects like repeat, cover or set position.
  • The img tag just places the pictures, it doesn't control the behaviour.

Other CSS properties that affect the display of background images

  • background-image: sets one or more image to background.
  • background-repeat:It defines how the background image repeats. Example, "repeat" will repeatboth horizontally and vertically. "no repeat" will have no repition.
  • background-position:Controls the background image is placed inside the element.
  • background-size:Sets how large the background image should be.

Difference between dimentions and resolution in bitmap image

  • Dimention refers to the image's physical size in pixel(height X width), while resolution describes how densely those pixels are packed(pixels per inch or PPI), affecting print quality but not on-screen size.
